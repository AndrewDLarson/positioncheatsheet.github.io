Fixed position

The fixed position removes the element from the document flow. No space is created in the document for the element. The element is pinned to that exact spot on the page. The element will stay the same regardless of scroling. You must then position the rest of the page below the fixed element or they will overlap. This can be done with position: relative;

EXAMPLE: is shown in header. It stays stationary as you scroll the page.

absolute position

Positions an element in a parent element that has the position relative or absolute. Ignores all sibling elements, so it is possible they overlap if they are not centered correctly.

EXAMPLE The green square is positioned with absolute inside its parent element, the purple square. I can move the green square around inside the purple square by adjusting the top, right, left, and bottom.

Z-index property

The level that the element is on. Almost like paper overlaping, 0 is the lowest and 10 is on top.

EXAMPLE: This box has a z-index of 1, while the orange box has a z-index of 0.

display property

Determines the type of block an element will have. Elements can have: block, inline, and inline-block displays

EXAMPLE: The blue explenation boxes and the green example boxes have inline-box display. meaning they are boxes that can be on the same line.

The heading "Display and Positioning Cheatsheet" would be considered a block display taking up its line.

The "example:" in this square is an inline display, as it can stay in a line of other text.

float property

An element is floated to the left or right and other elements will wrap around it.

EXAMPLE: This set of text wraps around the example tag. In order for it to wrap around though, it would have to be long enough to take up all the space on the right side and then continue to take up space underneath.
